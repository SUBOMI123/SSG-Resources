  <what-built>
N/A — Requires GitHub + Vercel account linking that Claude cannot complete.
  </what-built>
  <how-to-verify>
Follow these steps:

1. **Sign up for Vercel:**
   - Visit https://vercel.com
   - Click "Sign up" and choose "Continue with GitHub"
   - Authorize Vercel to access your GitHub repositories
   
2. **Import project:**
   - In Vercel dashboard, click "New Project"
   - Select "Import Git Repository"
   - Search for and select your "SSG-Resources" repository
   - Click "Import"

3. **Configure environment variables:**
   - In project settings, go to "Environment Variables"
   - Add these variables (mark as "Production" for auto-deploy):
     - Name: `DATABASE_URL`
       Value: (Paste Supabase connection string from Task 1)
       Environments: Select "Production"
   - Click "Save"

4. **Configure Node.js and build settings:**
   - Framework: "Next.js" (auto-detected)
   - Build command: `npm run build` (auto-detected)
   - Output directory: `.next` (auto-detected)
   - Install command: `npm install` (auto-detected)
   - Click "Deploy"

5. **Wait for initial deployment:**
   - Vercel will run `npm install` and `npm run build`
   - This may take 2-3 minutes
   - Once complete, you'll get a deployment URL

6. **Verify deployment:**
   - Click the deployment URL to test
   - Visit `/api/inventory` endpoint (should work if Phase 1 deployed)
   - Should see "500" or proper error (database not seeded yet is okay for Phase 0)

Expected outcome: Vercel dashboard shows "Deployment Successful" with a live URL

  <name>Task 3: Create vercel.json Configuration</name>
  <files>vercel.json</files>
  <action>
Create vercel.json at project root with deployment settings:

```json
{
  "buildCommand": "npm run build",
  "env": {
    "DATABASE_URL": "@database_url",
    "NODE_ENV": "production"
  },
  "git": {
    "deploymentEnabled": {
      "main": true
    }
  }
}
```

This ensures:
- Vercel uses `npm run build` (explicit, no guessing)
- Environment variables are sourced from Vercel dashboard
- Only main branch auto-deploys to production
- Feature branches get preview deployments
